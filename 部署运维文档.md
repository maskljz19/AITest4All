## 部署概述

### 部署方式

本系统支持两种部署方式：

1. **Docker 部署（推荐）**: 使用 Docker Compose 一键部署所有服务，适合快速部署和生产环境
2. **手动部署**: 分别部署各个组件，适合需要精细控制的场景

### 部署架构

```
                    ┌─────────────┐
                    │   Nginx     │
                    │  (反向代理)  │
                    └──────┬──────┘
                           │
              ┌────────────┴────────────┐
              │                         │
        ┌─────▼─────┐            ┌─────▼─────┐
        │  前端应用  │            │  后端API   │
        │  (React)  │            │ (FastAPI)  │
        └───────────┘            └─────┬──────┘
                                       │
                    ┌──────────────────┼──────────────────┐
                    │                  │                  │
              ┌─────▼─────┐      ┌────▼────┐      ┌─────▼─────┐
              │PostgreSQL │      │  Redis  │      │    LLM    │
              │  (数据库)  │      │ (缓存)  │      │   API     │
              └───────────┘      └─────────┘      └───────────┘
```

### 服务器要求

**最低配置**:
- CPU: 4核

**推荐配置**:

## 环境变量配置说明

### 项目中的三个 .env 文件

本项目包含三个 `.env.example` 配置文件模板，用途不同：

| 文件位置 | 用途 | 使用场景 |
|---------|------|---------|
| **`.env.docker.example`** (根目录) | Docker Compose 部署配置 | **Docker 部署时使用** |
| `backend/.env.example` | 后端应用配置 | 本地开发或手动部署后端时使用 |
| `frontend/.env.example` | 前端应用配置 | 本地开发或手动部署前端时使用 |

### Docker 部署时的配置

**重要**: Docker 部署时，**只需要配置根目录的 `.env` 文件**，不需要配置 `backend/.env` 和 `frontend/.env`。

原因：
- Docker Compose 会自动读取根目录的 `.env` 文件
- 环境变量会通过 `docker-compose.yml` 传递给所有容器
- 容器内的应用直接使用这些环境变量

### 手动部署时的配置

手动部署时，需要分别配置：
- `backend/.env` - 后端应用配置
- `frontend/.env` - 前端应用配置

## Docker部署

Docker 部署是推荐的部署方式，可以快速启动所有服务，无需手动安装和配置各个组件。

### 1. 安装Docker

```bash
# 安装Docker
curl -fsSL https://get.docker.com | sh

# 启动Docker
sudo systemctl start docker
sudo systemctl enable docker

# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

### 2. 配置环境变量

```bash
# 复制环境变量示例文件到根目录
cp .env.docker.example .env

# 编辑环境变量
vim .env
```

**必须配置的参数**:

```bash
# 数据库密码（必须修改）
POSTGRES_PASSWORD=your_secure_password_here

# LLM API 密钥（至少配置一个）
OPENAI_API_KEY=sk-your-openai-api-key-here
# 或
ANTHROPIC_API_KEY=your-anthropic-api-key-here

# 加密密钥（生产环境必须配置）
# 生成方法：python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
ENCRYPTION_KEY=your_base64_encryption_key_here
```

**可选配置参数**:

```bash
# 自定义 API 端点（使用代理或自建模型时）
OPENAI_API_BASE=https://api.openai.com/v1
ANTHROPIC_API_BASE=https://api.anthropic.com

# 默认模型配置
DEFAULT_MODEL_PROVIDER=openai
DEFAULT_MODEL_NAME=gpt-4
DEFAULT_TEMPERATURE=0.7
DEFAULT_MAX_TOKENS=2000

# 应用性能配置
APP_WORKERS=4  # 根据服务器 CPU 核心数调整
SESSION_EXPIRE_HOURS=24
MAX_UPLOAD_SIZE_MB=10
SCRIPT_TIMEOUT_SECONDS=30
LOG_LEVEL=INFO

# CORS 配置（生产环境添加你的域名）
CORS_ORIGINS=http://localhost,http://localhost:80,https://your-domain.com
```

**注意事项**:
- `.env` 文件已在 `.gitignore` 中，不会被提交到版本控制
- 生产环境务必使用强密码
- 生成唯一的加密密钥
- 根据实际域名配置 CORS

### 3. 启动服务

```bash
# 构建并启动所有服务
docker-compose up -

# 执行数据库迁移
docker exec ai-test-case-backend alembic upgrade head

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

服务启动后，Docker Compose 会自动：
- 创建并启动 PostgreSQL、Redis、Backend、Frontend 容器
- 创建数据卷用于持久化存储
- 配置容器间的网络连接
- 执行健康检查

### 4. 验证部署

访问以下地址验证服务是否正常：

- **后端 API**: http://your-server-ip:8000

- **健康检查**: http://your-server-ip:8000/health

- `postgres_data`: PostgreSQL 数据库数据

- `backend_uploads`: 用户上传的文件
- `backend_knowledge`: 知识库文件
- `backend_logs`: 应用日志

查看数据卷：

### 7. 常见问题

**Q: 为什么不需要配置 backend/.env 和 frontend/.env？**

A: Docker Compose 会将根目录 `.env` 文件中的环境变量传递给所有容器，容器内的应用直接使用这些环境变量。

**Q: 如何修改配置后重启服务？**

```bash
# 修改 .env 文件后
vim .env

# 重启受影响的服务
docker-compose restart backend

# 或重启所有服务
docker-compose restart
```

**Q: 数据存储在哪里？**

数据存储在 Docker 命名卷中，位置通常在 `/var/lib/docker/volumes/`。使用 `docker-compose down -v` 会删除数据卷，请谨慎操作。

**Q: 如何查看详细日志？**

```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend

# 查看最近 100 行日志
docker-compose logs --tail=100 backend
```

## 手动部署

手动部署适合需要精细控制各个组件的场景。手动部署时需要分别配置 `backend/.env` 和 `frontend/.env`。

### 1. 克隆代码

```bash
cd /var/www
sudo git clone <repository-url> ai-test-case-generator
cd ai-test-case-generator
```

### 2. 部署后端

```bash
cd backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
vim .env

# 初始化数据库
alembic upgrade head

# 测试启动
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 3. 部署前端

```bash
cd ../frontend

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env
vim .env

# 构建生产版本
npm run build

# 构建产物在 dist/ 目录
```

### 4. 配置系统服务

创建后端服务文件:

```bash
sudo vim /etc/systemd/system/ai-test-backend.service
```

内容:

```ini
[Unit]
Description=AI Test Case Generator Backend
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/ai-test-case-generator/backend
Environment="PATH=/var/www/ai-test-case-generator/backend/venv/bin"
ExecStart=/var/www/ai-test-case-generator/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务:

```bash
sudo systemctl daemon-reload
sudo systemctl start ai-test-backend
sudo systemctl enable ai-test-backend
sudo systemctl status ai-test-backend
```

## Nginx配置

### 1. 创建Nginx配置文件

```bash
sudo vim /etc/nginx/sites-available/ai-test-case-generator
```

配置内容:

```nginx
# 后端API服务器
upstream backend {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name your-domain.com;
    
    # 前端静态文件
    location / {
        root /var/www/ai-test-case-generator/frontend/dist;
        try_files $uri $uri/ /index.html;
    }
    
    # 后端API
    location /api/ {
        proxy_pass http://backend/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # WebSocket
    location /ws/ {
        proxy_pass http://backend/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        root /var/www/ai-test-case-generator/frontend/dist;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 2. 启用配置

```bash
# 创建软链接

# 测试配置

```

```nginx
gzip_types text/plain text/css text/xml text/javascript application/json application/javascript application/xml+rss;

# 缓存配置
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=1g inactive=60m;
```

## 故障排查

### 常见问题

#### 1. 后端服务无法启动

检查日志:

```bash
sudo journalctl -u ai-test-backend -n 100
```

常见原因:
- 端口被占用
- 数据库连接失败
- 环境变量配置错误
- Python依赖缺失

#### 2. 数据库连接失败

检查PostgreSQL状态:

```bash
sudo systemctl status postgresql
```

测试连接:

```bash
psql -U aitest -h localhost -d ai_test_case_generator
```

#### 3. Redis连接失败

检查Redis状态:

```bash
sudo systemctl status redis-server
redis-cli ping
```

#### 4. Nginx 502错误

检查后端服务:

```bash
sudo systemctl status ai-test-backend
curl http://localhost:8000/health
```

检查Nginx配置:

```bash
sudo nginx -t
```

#### 5. 磁盘空间不足

查看磁盘使用:

```bash
df -h
du -sh /var/* | sort -h
```

清理日志:

```bash
sudo journalctl --vacuum-time=7d
sudo find /var/log -name "*.log" -mtime +30 -delete
```

## 安全加固

### 1. 防火墙配置

```bash
# 安装UFW
sudo apt install -y ufw

# 允许SSH
sudo ufw allow 22/tcp

# 允许HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status
```

### 2. 配置SSL/TLS

使用Let's Encrypt:

```bash
sudo certbot --nginx -d your-domain.com
```

### 3. 数据库安全

```bash
# 修改PostgreSQL密码
sudo -u postgres psql
ALTER USER aitest WITH PASSWORD 'new_secure_password';

# 限制远程访问
sudo vim /etc/postgresql/14/main/pg_hba.conf
# 只允许本地连接
host    all             all             127.0.0.1/32            md5
```

### 4. Redis安全

```bash
sudo vim /etc/redis/redis.conf

# 设置密码
requirepass your_redis_password

# 禁用危险命令
rename-command FLUSHDB ""
rename-command FLUSHALL ""
rename-command CONFIG ""
```

- 定期更新依赖包
- 使用强密码
- 启用HTTPS
- 配置CORS
- 实施访问控制
- 定期备份数据
